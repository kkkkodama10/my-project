# 機能追加セットアップ

このコマンドは、既存リポジトリに新機能を追加するための3つの設計ドキュメントを対話的に作成します。

## 実行方法

```bash
claude
> /setup-dev 20260401_awesome-feature
```

**引数:** 機能ディレクトリ名（`ideas/` 配下のディレクトリ名と同じ形式: `YYYYMMDD_<feature-name>`）

## 実行前の確認

`ideas/[引数]/` ディレクトリ内のファイルを確認します。

```bash
# 確認
ls ideas/[引数]/

# ファイルが存在する場合
✅ ideas/[引数]/ にアイデアが見つかりました
   この内容を元に設計ドキュメントを作成します

# ファイルが存在しない場合
⚠️  ideas/[引数]/ にファイルがありません
   対話形式で要求定義を作成します
```

## 手順

### ステップ0: インプットの読み込み

1. `ideas/[引数]/` 内のマークダウンファイルを全て読む
2. 既存リポジトリのコードベースをGrepで調査し、現在のアーキテクチャ・技術スタックを把握する
3. `docs/[引数]/` ディレクトリを作成する

### ステップ1: 要求定義書の作成

1. **requirements-writingスキル**をロード
2. `ideas/[引数]/` の内容とステップ0で把握した既存コードベースの情報を元に、スキルのテンプレートを参考に `docs/[引数]/requirements.md` を作成
3. **既存機能への影響・互換性を明確に記述する**
4. ユーザーに確認を求め、**承認されるまで待機**

### ステップ2: 機能設計書の作成

1. **functional-designスキル**をロード
2. `docs/[引数]/requirements.md` を読む
3. 既存コードベースの関連部分をGrepで調査
4. スキルのテンプレートを参考に `docs/[引数]/functional-design.md` を作成
5. **既存コードとの接続点を明確に記述する**
6. ユーザーに確認を求め、**承認されるまで待機**

### ステップ3: アーキテクチャ設計書の作成

1. **architecture-designスキル**をロード
2. 既存のドキュメントと既存コードベースを読む
3. スキルのテンプレートを参考に `docs/[引数]/architecture.md` を作成
4. **既存アーキテクチャからの変更差分を重点的に記述する**:
   - 変更が必要なレイヤー・コンポーネント
   - 新規追加するファイル・ディレクトリ
   - 既存コードへの影響範囲
   - 依存関係の変更
5. ユーザーに確認を求め、**承認されるまで待機**

## 完了条件

- 3つの設計ドキュメントが全て作成されていること

完了時のメッセージ:
```
「設計ドキュメントの作成が完了しました!

作成したドキュメント:
✅ docs/[引数]/requirements.md
✅ docs/[引数]/functional-design.md
✅ docs/[引数]/architecture.md

今後の使い方:
- 機能の実装: /add-feature [機能名] を実行してください
- ドキュメントレビュー: /review-docs [パス] を実行してください
」
```
