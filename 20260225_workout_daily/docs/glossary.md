# ユビキタス言語定義

プロジェクト内で使う用語の意味を統一する。ドキュメント・コード・会話すべてでこの定義に従う。

## ドメイン用語

| 用語 | 定義 |
|------|------|
| **健康日記** | GarminのヘルスデータをもとにClaude APIが生成する、1日分の自然文テキスト |
| **ヘルスデータ** | Garmin Connectに記録された1日分の生体・活動データ（睡眠・歩数・ストレス）の総称 |
| **対象日付** | 日記を生成する対象の日付。デフォルトは「昨日」（実行日の前日） |
| **欠損データ** | Garmin Connectから取得できなかったデータ項目。`None` で表現される |
| **自然文** | 数値の羅列でなく、人が読んで自然に感じる日本語の文章 |

## 睡眠データ用語

| 用語 | 定義 |
|------|------|
| **合計睡眠時間** | 深い睡眠・浅い睡眠・REMの合計時間（時間単位、小数） |
| **深い睡眠** | ノンレム睡眠のうち最も深い段階（deep sleep） |
| **浅い睡眠** | ノンレム睡眠のうち浅い段階（light sleep） |
| **REM睡眠** | 急速眼球運動を伴う睡眠段階。夢を見やすい |

## ストレスデータ用語

| 用語 | 定義 |
|------|------|
| **ストレスレベル** | Garminが心拍変動(HRV)から算出するストレス指標。0〜100の数値 |
| **ストレス平均** | 1日を通じたストレスレベルの平均値 |

## システム用語

| 用語 | 定義 |
|------|------|
| **Garmin Connect** | Garminデバイスのデータが同期されるクラウドサービス |
| **セッショントークン** | Garmin Connectへの認証後に発行されるトークン。`~/.garminconnect` にキャッシュして再利用する |
| **health_data** | `GarminClient` が返す内部データ形式。Pythonのdict型。詳細は `docs/architecture.md` §4.1 を参照 |
| **Notionデータベース** | 日記ページを格納するNotionのデータベース。`NOTION_DATABASE_ID` で指定する |
| **フォールバック** | Notion APIへの投稿が失敗したとき、ローカルにMarkdownとして保存する代替処理 |

## ドキュメント用語

| 用語 | 定義 |
|------|------|
| **永続的ドキュメント** | `docs/` 配下のドキュメント。プロジェクト全体の設計・方針を定義し、長期間維持される |
| **作業単位のドキュメント** | `.steering/` 配下のドキュメント。1タスクのスコープで作成・破棄される |
| **MVP** | Minimum Viable Product。Phase 1における最小成功条件。CLI実行で日記テキストが生成されること |
| **Phase** | 開発フェーズ。Phase 1〜3で機能を段階的に拡張していく（詳細は `docs/product-requirements.md` §5）|
