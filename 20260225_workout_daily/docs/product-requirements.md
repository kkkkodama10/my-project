# プロダクト要求定義書

## 1. プロダクト概要

**プロダクト名**: Auto Diary
**一行説明**: Garminのヘルスデータから自然文の健康日記を毎日自動生成するPythonアプリケーション

## 2. 背景と課題

- 健康データはGarminで記録されているが、数値の羅列のまま放置されがち
- 毎日手動で日記を書くのは続かない
- データを「読み返せる文章」として蓄積したい

## 3. ユーザー

- **対象**: 自分自身（個人利用・シングルユーザー）
- **前提**: Garminデバイスを日常的に装着し、Garmin Connectにデータが同期されている

## 4. ゴール

ユーザーが何もしなくても、毎日「今日の健康日記」が自動で作成される。

## 5. フェーズ計画

| Phase | 内容 | 状態 |
|-------|------|------|
| **Phase 1** | Garminデータ（睡眠・歩数・ストレス）→ Claude APIで日記生成 | 着手中 |
| Phase 2 | 写真のExifデータ（位置・時刻）を加えて「行動日記」に拡張 | 未着手 |
| Phase 3 | デバイス使用時間を追加し「知的活動」も含めた完全版 | 未着手 |

## 6. Phase 1 スコープ

### 6.1 MVP（最小成功条件）

CLIで `python src/main.py` を実行すると、Garminデータを取得し、やわらかい自然文の日記テキストが生成される。

- Notion投稿はMVP後に追加する（Phase 1内だがMVPには含めない）
- cron自動実行もMVP後に設定する

### 6.2 機能要件

| # | 機能 | 優先度 | MVP |
|---|------|--------|-----|
| F1 | Garmin Connectから睡眠・歩数・ストレスデータを取得 | 必須 | o |
| F2 | 取得データをClaude APIに渡して自然文の日記を生成 | 必須 | o |
| F3 | 生成された日記をNotionデータベースに投稿 | 必須 | - |
| F4 | cronによる毎日の自動実行 | 任意 | - |

### 6.3 日記の品質要件

- **トーン**: やわらかい自然文（レポート調ではない）
- **数値の扱い**: 「約〇〇」「〇〇ほど」のように丸めて自然に表現
- **欠損データ**: 触れない（取得できたデータだけで構成する）
- **言語**: 日本語
- **長さ**: 2〜5文程度の短文

**出力イメージ**:
> 6時間半ほど眠り、深い睡眠は約1時間だった。8,400歩ほど歩き、ストレスレベルは比較的低く推移した。

### 6.4 データ要件

Garmin Connectから取得する対象データ（Phase 1）:

| データ | 主な値 |
|--------|--------|
| 睡眠 | 合計睡眠時間、深い睡眠時間、浅い睡眠時間、REM睡眠時間 |
| 歩数 | 合計歩数 |
| ストレス | 日平均ストレスレベル |

- 各データの取得は独立しており、一部取得失敗でも他のデータは利用する
- 全データ取得失敗の場合は日記を生成しない

## 7. 非機能要件

- **実行環境**: ローカルマシン（macOS）
- **認証情報**: `.env` ファイルで管理（Git管理外）
- **ログ**: 成功/失敗・取得データ項目・生成テキスト文字数を記録
- **エラー時**: リトライ1回。失敗時はログに記録して終了

## 8. スコープ外（Phase 1）

- Webインターフェース
- 複数ユーザー対応
- 日記の編集・削除機能
- 過去日記の振り返り・サマリー
- 写真・スクリーンタイムデータの統合
