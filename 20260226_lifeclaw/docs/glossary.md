# ユビキタス言語定義

本プロジェクトで使用する用語の意味を統一する。

## プロダクト用語

| 用語 | 定義 |
|------|------|
| ライフログ | ユーザーの日常（仕事・学習・睡眠・家族等）を構造化して記録したもの |
| パーソナルコンテキスト | 個人の思考・行動・学習の履歴。AI活用の基盤となるデータ |
| ログ整形 | ユーザーの雑なメモをLLMで日報フォーマット（構造化JSON）に変換する処理 |
| 振り分け | メモの内容をNotionの各プロパティ（仕事・勉強・メモ等）に分類すること |
| 追記 | 同一日付のNotionレコードに対し、既存の内容を保持したまま新しい内容を末尾に追加すること |
| リマインド | 毎日定時にLINEで送信する記録促進の通知。固定メッセージでLLM呼び出しなし |
| 元メッセージ | ユーザーがLINEで送信した原文テキスト。整形前の状態でNotionに保存する |

## システム用語

| 用語 | 定義 |
|------|------|
| OpenClaw | 自分のデバイスで動かすパーソナルAIアシスタント（OSS）。npmパッケージ（Node ≥22）。Gateway・スキル・cronの実行基盤 |
| Gateway | OpenClawのWebSocket制御プレーン（port 18789）。チャネル接続、Webhook受信、スキル実行を統括する |
| スキル | OpenClawの拡張単位。`~/.openclaw/workspace/skills/<skill>/SKILL.md` で定義されるプロンプトとツール許可の組み合わせ |
| lifelogスキル | 本プロジェクト用の自作スキル。メモ受信→LLM整形→Notion書き込みを行う |
| バンドルスキル | OpenClawに同梱される公式スキル。本プロジェクトでは `notion` のみ許可 |
| Tailscale Funnel | TailscaleのHTTPSトンネル機能。ローカルポートをインターネットに安全に公開する |
| isolatedセッション | OpenClawのcronジョブ実行時に使用する独立セッション。メインの会話コンテキストを汚さない |
