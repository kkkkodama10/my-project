# 要求定義書作成スキル

このスキルは、既存リポジトリに追加する新機能の要求定義書を作成するためのガイドです。

## 前提条件

### アイデアメモが存在する

**推奨**: `ideas/YYYYMMDD_<feature-name>/` にアイデアメモが配置されていること。

存在しない場合は、対話形式で要求を整理します。

### 既存コードベースの把握

要求定義書作成前に、既存コードベースを検索して:
- 現在の機能・画面構成
- 関連する既存機能
- 技術スタック・制約

を理解してください。

## 出力先

```
docs/YYYYMMDD_<feature-name>/requirements.md
```

## 作成手順

### 1. アイデアの読み込み

`ideas/YYYYMMDD_<feature-name>/` 内のマークダウンファイルを全て読む。

### 2. 既存システムの調査

既存コードベースを検索し、以下を把握:
- 現在の機能・ユースケース
- 新機能が影響する既存部分
- 技術的な制約

### 3. 要求定義書の作成

テンプレートに従って作成。以下の観点を重視:

**既存リポジトリ固有の観点**:
- 新機能が既存機能とどう連携するか
- 既存ユーザー体験への影響
- 既存データとの互換性
- 段階的リリースの可能性

### 4. セルフレビュー

作成後、以下を確認:
- [ ] 解決する問題が具体的か
- [ ] 受け入れ条件が検証可能か
- [ ] スコープが明確か（やること/やらないこと）
- [ ] 既存機能への影響が考慮されているか
- [ ] 成功指標が測定可能か

## テンプレートの参照
