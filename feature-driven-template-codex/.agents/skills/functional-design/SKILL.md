# 機能設計書作成スキル

このスキルは、既存リポジトリに追加する新機能の機能設計書を作成するためのガイドです。

## 前提条件

### 要求定義書が作成されている

**必須**: 該当機能の要求定義書が存在する必要があります:

**ファイルパス**: `docs/YYYYMMDD_<feature-name>/requirements.md`

### 既存コードベースの把握

機能設計書作成前に、既存コードベースを検索して:
- 現在のアーキテクチャパターン
- 関連する既存コンポーネント
- 命名規則・コーディングスタイル

を理解してください。

## 既存ドキュメントの優先順位

**重要**: 該当機能の機能設計書がすでに存在する場合:

1. **既存の機能設計書** - 最優先
2. **このスキルのガイド** - 参考資料（新規作成時または補足として使用）

## 出力先

```
docs/YYYYMMDD_<feature-name>/functional-design.md
```

## 重要: 既存コードとの接続

新規プロジェクトと異なり、既存リポジトリへの機能追加では以下を重点的に記述してください:

- **既存コンポーネントとの接続点**: どの既存クラス/関数/モジュールと連携するか
- **既存データモデルとの関係**: 既存テーブル/スキーマへの変更・拡張
- **既存APIとの整合性**: エンドポイント命名・レスポンス形式の統一
- **影響範囲**: 既存機能への副作用
